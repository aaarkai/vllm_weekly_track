# Weekly Release Report for vllm-project/vllm (2026-06-19)

This week merged 278 PRs from 153 contributors. Key areas: features 3, fixes 70, performance 22.

## Executive Summary

本周主要更新聚焦于模型支持、性能优化与大量缺陷修复。特性方面，新增DiffusionGemma模型支持，并引入Streaming Parser Engine及Qwen3解析器。性能工作涵盖DeepSeek-OCR的双路径ViT CUDA图支持、DSv4的Flashinfer稀疏索引缓存（带来2%~4%的TTFT提升），以及ROCm硬件上对Minimax-M3的FP8 per_channel优化与W4A16 FlyDSL MoE支持。此外，Marlin内核迁移至torch stable ABI，Triton重编译检测功能也已加入。

缺陷修复涉及多个重点领域：解析器修复了Qwen3参数截断、Gemma4流式推理状态初始化及工具调用问题；MoE模型修复了路由输出未填充与FP8 LoRA输出损坏；量化方面修正了FP8 KV缓存误拒、E8M0缩放计算错误等；ROCm修复了自定义操作符别名；前端修复了FastAPI路由缺失、指标未遵循服务模型名等。另有多个CI与构建问题被解决，包括Docker依赖图、调度器测试、SD LoRA等。本周期无破坏性变更或特殊升级指引。
