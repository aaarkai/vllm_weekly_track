# Weekly Release Report for vllm-project/vllm (2026-05-01)

This week merged 193 PRs from 124 contributors. Key areas: features 4, fixes 50, performance 11.

## Executive Summary

本周期发布重点包括多项新功能、大规模缺陷修复、性能优化与模型扩展。功能层面，vLLM Chat Completions API 新增 `prompt_embeds` 内容部分支持，并引入 `system_fingerprint` 字段；新增 Moondream3 模型支持（仅 query 与 caption 技能）以及 humming 量化内核；硬件支持方面，为 CPU 添加了基于 AMX/AVX-512 的 FP8 attention，并为 ViT 实现了 FP8 FlashInfer attention，同时新增 nvfp4 KV cache 与 H100 上基于 triton 的 fused_moe 配置。

缺陷修复覆盖广泛，涉及 Gemma4 MoE、DeepSeek V3/V4、Qwen3、MLA 等多个模型及组件，包括 KV 块分配失败、OOM、deadlock、token 泄漏、权重映射错误等问题。性能优化方面，`AllPool.forward` 提速约 51%，FlashInfer top-k/top-p 采样器默认启用，并加入 FlashInfer FP8 异步 TP fusion 与 NVFP4 dequantization Triton 内核。新增模型包括 Laguna XS.2、MiMo-V2.5 以及 Cohere MoE。破坏性变更方面，仅涉及 CI 跳过 terratorch 与 torchgeo 测试，对下游用户无直接影响。建议升级前注意验证与 FlashInfer 及量化相关配置的兼容性。

## Upgrade Notes

- The upgrade from PyTorch 2.9 to 2.10 seems to have substantially increased memory fragmentation during startup and cudagraph capture, leading to OOMs during startup for some models at default --gpu-memory-utilization (e.g. GPT-OSS 120B TP2 on 2xH100).
