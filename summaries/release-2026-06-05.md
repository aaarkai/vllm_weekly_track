# Weekly Release Report for vllm-project/vllm (2026-06-05)

This week merged 236 PRs from 134 contributors. Key areas: features 5, fixes 39, performance 24.

## Executive Summary

本周的版本更新主要集中在核心内核迁移、新模型支持、性能优化与大量 Bug 修复方面。**Features** 方面，继续推进 attention 与 cache 内核向 torch stable ABI 的迁移，新增 Rust Frontend 的动态 LoRA 端点，并支持 Granite Speech Plus、ModelOpt MXFP8 非门控 MoE 等模型。**Fixes** 覆盖了 Gemma4 系列（包括 MTP 与 Unified 模型）的多项启动、内存访问及精度问题，修复了 MiniCPM-V 视频推理崩溃、DeepSeek V4 初始化错误、LoRA 适配器查找错误、工具调用参数丢失等关键缺陷，以及 NCCL 通信、KV cache 形状、prefix cache 损坏等问题。**Performance** 方面，优化了 cutlass fp8 scaled mm 内核（性能提升 20%）、Hopper 上默认使用 triton MoE 后端、多模态 item 处理从 O(n) 降低至 O(log n)，并新增 H200 与 RTX PRO 的 tuned selective_state_update 配置、ViT 全 CUDA graph 支持 InternVL、Nixl 零拷贝通信等。**Model/Hardware** 方面，新增 Gemma4 Unified（无编码器）支持、JetBrains Mellum v2 模型，并在 ROCm 侧优化了 Docker 构建缓存、W4A16 内核与 workspace 管理。本次发布无明确 breaking changes 或升级指引。

## Upgrade Notes

- Continues the libtorch stable ABI migration by moving several kernels out of legacy _C and into _C_stable_libtorch.
- Migration progress using the Audit Python extension [torch-abi-audit](https://pypi.org/project/torch-abi-audit/):
- **Note:** because of the lack of support in torch 2.10 for the version-guarded `deleter` in `get_cuda_view_from_cpu_tensor`, there were many updates to the file `cuda_view.cu` causing GitHub to treat is as a deleted/created file instead of a moved file. To check the diffs, look at this commit: https
- Vendors `MiniCPMVProcessor` and `MiniCPMOProcessor` into vLLM to unblock the Transformers v5 upgrade.
- Note: the CI image still emits the existing duplicate CuPy warning (`cupy-cuda12x`, `cupy-cuda13x`) after connector installation. This PR leaves that as a separate follow-up unless it remains an active connector-job failure after the NIXL wheel fix.
- Upgrade AITER to v0.1.13.post1 for kimi-k2.5 perf
- * **The impact:** With this small change, we've fixed 8 failures in `tests/compile/test_dynamic_shapes_compilation.py` (specifically in the unbacked variants for Qwen2-7B and Llama-3.1-8B) that were breaking our AMD CI, which runs on torch 2.10.x.
