# Weekly Release Report for vllm-project/vllm (2026-06-26)

This week merged 292 PRs from 150 contributors. Key areas: features 2, fixes 72, performance 20.

## Executive Summary

本周主要围绕特性增强、性能优化与大量Bug修复展开。特性方面，前端将ServingRender拆分为独立的renderer与entrypoint，新增对OpenMOSS-Team模型的支持，并为MiniMax-M3引入modelopt nvfp4量化及fp8 per-block在线量化的权重填充功能。性能优化重点覆盖低延迟场景：新增cluster-cooperative topK kernel（DSv4/DSv3.2），在ROCm上优化了minimax-m3稀疏注意力，并在AArch64 CPU上加速未量化MoE。此外，多个MoE内核获得改进（矢量化fp32 moe_sum reduction、低batch decode的块FP8 fused MoE调优），FlashInfer MXINT4 MoE新增gated SiLU支持，LoRA的convert_mapping以字典替代线性搜索提升效率，并跳过DP填充token。

修复方面，大量问题得到解决，涵盖模型Runner V2（mamba hybrid模型seq_lens_cpu_upper_bound转发、sampler kernel int32溢出等）、ROCm（HIP fork重初始化、上下文并行与TRITON_MLA块对齐、cumem休眠与拆卸）、前端（Anthropic空补全缺失内容块、非ASCII工具调用转义、空tool_calls省略）、工具解析器（PoolsideV1空白与命名工具选择、流式推理标记解析）、模型加载与量化（Llama4初始化、NVFP4+MTP崩溃、默认tie_weights绑定权重）、KV offload生命周期与竞争条件、DeepEP V2边界约束、安全（split_audio NaN无限循环）、MoE FP8 LoRA响应损坏及压缩张量WNA16 MoE静态actorder处理等。本周未涉及重大Breaking变更或升级指引，重点在于稳定性和多平台兼容性提升。
