# Weekly Release Report for vllm-project/vllm (2026-07-24)

This week merged 224 PRs from 128 contributors. Key areas: features 3, fixes 59, performance 14.

## Executive Summary

本周发布版本集中于推理性能优化与大量稳定性修复，同时引入了新的 JIT 预热基础设施及 MoE 量化方案迁移。性能方面，针对 Blackwell 的 decode 阶段优化（GLM-5.2）和 DeepSeek-V4 的路由内核专精优化带来显著端到端加速；新增的 JIT 预热框架有望减少冷启动开销；MoE 量化方法 MoeWNA16 已迁移至新的 MK oracle 方案以提升调度效率。修复方面覆盖广泛，包括混合精度 KV 缓存检测、DeepGEMM 在 FlashInferFp8 内核下的预热失败、MLA 在 FlashAttention 4 JIT 预热中的降级处理、以及 Rust 前端对零列 logprobs 和缺失 prompt logprobs 的 panic 修复。多模态模型（如 Qwen3-VL、MiniCPM-V）的权重加载与像素边界处理得到修正，KV offloading 与 EAGLE draft 传播相关的多项竞态条件也得到解决。注意，部分修复涉及量化配置传播与 reload 后的参数同步，建议存在自定义量化或权重重载场景的用户关注对应 PR。此外，ROCm 平台持续获得 CI 镜像准备与稀疏 decode 占用率提升，AMD GPU 用户可留意相关改进。
